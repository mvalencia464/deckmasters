Goal: Create a second, high-performance reviews section for my homepage.
1. The Component: Create an astronomical based review component. It should look identical to my current Reviews.astro but will consume data from a new file: src/data/google-reviews.json.
2. The Data Fetcher: Write a Node.js script scripts/sync-reviews.js that:
* Hits the DataForSEO Business Data API (Google Reviews endpoint).
* Performance Obsession: Downloads the profile_image_url avatars and saves them locally to public/images/avatars/. Update the JSON to point to these local paths. This prevents "Third-party request" penalties in Lighthouse.
* Maps the DataForSEO response to match the key names in my existing MoreGoodReviews JSON so I don't have to rewrite the CSS.
3. Performance Constraints:
* Zero JS: The new component must be 100% HTML/CSS.
* Keep same optimization as we have because it is perfect as is for speed, performance, etc. 
* Implementation: > - Add this new component right below our current one.
